#### 1、BaseGridView.FOCUS_SCROLL_PAGE 

`focusScrollStrategy = BaseGridView.FOCUS_SCROLL_PAGE`
配置成翻页效果，有各种奇怪问题：

a、如果一直按住遥控器的方向键不放，程序多半会崩溃
    解决办法：在dispatchKeyEvent方法或者在unhandledKeyListener等中判断当时列表是否在滚动（scrollState != RecyclerView.SCROLL_STATE_IDLE则可以认为在滚动）
    ，如果在滚动则消耗掉事件
    
```
override fun dispatchKeyEvent(event: KeyEvent): Boolean {
        //bugfix：修正控件处于遥控器按键一直按下时，引起列表快速滑动或翻页引起NPE问题导致崩溃
        //NullPointerException androidx.leanback.widget.GridLayoutManager$2.getCount(GridLayoutManager.java:1622) - Google 搜索
        if (this.isScrolling()) {x
            return true
        }

        return super.dispatchKeyEvent(event)
    }
```

b、场景：VerticalGridView设置成Page模式，350dp高度，item高度=50，刚好显示7个。发现按遥控器down，列表能正常翻页，
按遥控器上键能翻页但是总是会显示异常，不然当前位置是21，往上按翻页，则应该显示14 15 16 17 18 19 20，并选中20，但是实际情况显示的是13-19，选中的是20，但是20这个item并不可见。


#### 2、scrollToPosition
发现focusScrollStrategy设置为BaseGridView.FOCUS_SCROLL_PAGE或BaseGridView.FOCUS_SCROLL_ITEM时，列表不会滚动到指定位置，只有FOCUS_SCROLL_ALIGNED时才会滚动到指定位置。
